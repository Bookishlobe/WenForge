## 三、架构备忘

### 目录结构
```
wenforge/
├── src/                    # React 前端
│   ├── components/         # 组件
│   ├── hooks/              # 自定义 hooks
│   ├── stores/             # Zustand 状态管理
│   ├── lib/                # API 调用层
│   └── types/              # TypeScript 类型
├── python/                 # Python 后端
│   ├── main.py             # FastAPI 入口
│   ├── auto_writer.py      # 自动写作引擎
│   ├── config.py           # 配置管理
│   ├── storage.py          # 文件存储
│   ├── providers/          # AI 提供商适配器
│   ├── innovation/         # 灵犀引擎
│   ├── fact_engine/        # 事实断言引擎（后端已完成）
│   └── pipeline/           # 流水线引擎（后端已完成）
├── electron/               # Electron 主进程
└── docs/                   # 文档
```

### 数据流
```
用户操作 → React 组件 → Zustand Store → api.ts → IPC bridge
    ↓                                                    ↓
  Python FastAPI sidecar (127.0.0.1:8765) ← ← ← ← ← ← ←
    ↓
  AI Provider API / 本地文件系统
    ↓
  返回结果 → 回传到前端 → 更新 UI
```

### AI 提供商路由
```
writing_model   → 写作（经济模型，默认 deepseek-chat）
polish_model    → 润色（旗舰模型，默认 claude-sonnet-4-6）
outline_model   → 大纲（默认同 writing_model）
tier_a_model    → Pipeline A 级（旗舰）
tier_b_model    → Pipeline B 级（经济）
tier_c_model    → Pipeline C 级（本地）
```

## 四、已知问题

- 灵犀素材库使用 localStorage，更换设备/清除浏览器数据会丢失
- Python 后端重启后生成中的进度会丢失（需持久化生成状态）
- 无用户登录/多用户支持
- Electron 未打包为可分发的安装包
- 流水线和事实引擎缺乏前端界面
